# Claude Code - Projektanweisungen

## Arbeitsweise
- Arbeite selbstständig und eigenverantwortlich, ohne Rückfragen
- Triff Entscheidungen eigenständig — wähle den besten Weg
- Frage NUR bei destruktiven/irreversiblen Aktionen nach (z.B. force-push, Datenbank-Drop)
- Erkläre Entscheidungen kurz im Output, aber frage nicht um Erlaubnis
- Wenn mehrere Lösungswege existieren: wähle den besten und begründe kurz

## Code-Stil
- Bestehende Patterns und Konventionen im Projekt übernehmen
- Keine unnötigen Refactorings, Docstrings oder Kommentare hinzufügen
- Halte dich an KISS — kein Over-Engineering
- Sprache im Code und Commits: Englisch
- Kommunikation mit dem User: Deutsch

## Kontext-Management
- Auto-Compact ist aktiviert und darf laufen
- Ab ca. 70% Kontextauslastung: selbstständig /clear ausführen und mit frischem Kontext weiterarbeiten
- Vor einem /clear: aktuellen Fortschritt und nächste Schritte im Memory festhalten, damit nahtlos weitergearbeitet werden kann
- Den Zeitpunkt für /clear selbst bestimmen — idealerweise an einem natürlichen Pausenpunkt (nach Abschluss eines Teilschritts, nicht mitten in einer Änderung)

## Git
- Commits nur wenn explizit angefragt
- Commit-Messages auf Englisch, Conventional Commits Format
